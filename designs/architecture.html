Ingestion → Scoring Pipeline

Intake & Queue

02
Parse & Extract
PDF/DOCX parse. Extract skills, title, yoe, edu, location via NER. Structured JSON.
~2s / resume
03
Embed → ANN Index
Batch embed with text-embedding-3-small. Store in Weaviate / OpenSearch kNN. Pre-embed all JDs.
1536-dim vectors
04
ANN Pre-Screen
For each JD, ANN top-K (K=50) candidates. Cosine similarity gate ≥ 0.72. Kills 98% of pairs.
8K × 50 = 400K
05
Rule + LLM Score
Hard rules first (HK filter). Then LLM batch scoring with structured rubric. Weighted composite.
Haiku / prompt cache
06
Shortlist & Notify
Rank by score. Top-N per JD to recruiter dashboard. Webhook + email. Audit trail.

Cost Optimisation Levers
①
Prompt Caching
Cache the static scoring rubric prefix in Claude. Saves ~60% of LLM tokens on repeated batches.
②
ANN Kills 98% Pairs
Only 40K pairs reach LLM instead of 8B. Single biggest cost lever in the system.
③
Hard Rules Before LLM
Location, visa, YoE band, degree hard gates. Cuts LLM calls by another 30-40%.
④
Batch API + Spot Nodes
Use Anthropic Batch API (50% discount). AKS Spot for parse/embed workers.
⑤
Embed Cache TTL
Resume vectors cached 30 days. Re-embed only on CV update. JD vectors cached until JD changes.
Scalability & Resilience Design
Horizontal Scale
